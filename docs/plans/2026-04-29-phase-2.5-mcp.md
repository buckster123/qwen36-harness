# 2026-04-29 Phase 2.5 — MCP integration

> **Status:** drafted, not started. Pick up from here in next session.

**Goal:** Let the harness register tools from any external MCP (Model
Context Protocol) server transparently, so calling `filesystem-mcp`,
`github-mcp`, `brave-search`, etc. is identical to calling our builtin
tools. Massive leverage — instead of writing every integration ourselves,
we get the entire MCP ecosystem.

**Architecture:** A new `harness/mcp.py` module that:
1. Spawns one or more MCP servers as stdio subprocesses (configured in a
   new `configs/mcp_servers.toml`)
2. Speaks JSON-RPC 2.0 over their stdin/stdout per the MCP spec
3. On startup: calls `tools/list` on each server, registers each returned
   tool into our `default_registry` with a name like `<server>.<tool>`
   (e.g. `fs-mcp.read_file`)
4. Dispatch: when our registry resolves an MCP-backed tool, the wrapper
   sends `tools/call` JSON-RPC to the server, awaits, and returns the
   result.text content
5. Lifecycle: subprocess started lazily on first use, killed on shutdown

**Reference:** https://modelcontextprotocol.io/  ·  https://github.com/modelcontextprotocol

## Decisions to make at start of next session

1. **Where MCP-tool errors go.** MCP servers can return a structured
   error vs. our `ToolError` pattern. Recommend: pass through, mark
   `is_error=True`, append to `role: tool` content as text — same as
   builtin tools.
2. **Streaming MCP results.** MCP supports streaming tool results;
   our agent currently buffers tool output as a single string. Phase 2.5
   keeps it simple (buffer); revisit when we see a slow tool.
3. **Long-running tool processes.** Some MCP tools (e.g. shell-mcp
   running a build) take minutes. Need a cancellation hook. Decision:
   add a `cancel()` method on Agent that propagates to in-flight tool
   dispatch via `asyncio.Task.cancel()`. Phase 2.5 punt — note as TODO.

## Risk

- `mcp` SDK is new and changing. Pin to a specific version in pyproject.
- `uvx` / `npx` need to exist on the path; document in README.
- Stdio MCP servers buffer aggressively if their stdout isn't consumed
  fast enough; ensure we drain promptly.
